# Servo Bot: 対話型サーボ制御パラメータ調整アシストツール

本プロジェクトは、サーボ制御（PID制御）のシミュレーションと、AIアシスタントおよびベイズ最適化（Optuna）を用いたパラメータ調整支援を行うマルチページStreamlitアプリケーションです。

## 🛠 システム構成

本システムは以下の主要コンポーネントで構成されています。

1.  **Frontend (UI)**: Streamlitを使用したインタラクティブなマルチページ・インターフェース。
2.  **Simulation Engine**: サーボ系の物理挙動（1慣性・2慣性系）を精密に計算するシミュレータ。
3.  **AI Assistant**: LangChainとGemini/OpenAIを統合。専門知識および現在の波形データに基づき調整アドバイスを提供。
4.  **Auto Tuning**: Optunaを用いたPIDゲインの自動最適化。

## 📂 ディレクトリ構造と各ファイルの役割

### アプリケーション・コア
- `app.py`: **メインエントリポイント**。`st.navigation` を使用したマルチページ構成を定義します。
    - **対話型パラメータ調整Bot**: 手動調整、AIチャット、ベイズ最適化を含むフル機能ページ。
    - **自動パラメータ調整エージェント**: 今後エージェント機能を拡張予定の調整特化ページ。
- `README.md`: 本ドキュメント。

### `core/` (制御・データ中核)
- `state_manager.py`: セッション状態の一元管理（履歴、データ、UI状態）。
- `simulator_bridge.py`: UI層とシミュレーション・エンジン（`servo_sim.py`）を接続するインターフェース。
- `config.py`: 物理モデルの定数、制御パラメータのデフォルト値、データ構造定義。

### `services/` (高度な機能サービス)
- `llm_service.py`: 専門ドキュメント（RAG）を読み込み、コンテキストに応じたAIアドバイスを生成。
- `optimization_service.py`: Optunaを用いた探索アルゴリズムの実行。
- `analysis_service.py`: 統計計算およびグラフ描画処理のラップ。

### `views/` (UIコンポーネント)
- `sidebar.py`: 物理・制御パラメータ設定用サイドバー。
- `dashboard.py`: シミュレーション結果の可視化とメトリクス表示。
- `chat_ui.py`: AIアシスタントとのチャットインターフェース。
- `optimization_ui.py`: ベイズ最適化の実行制御と結果反映。

### 物理シミュレーション・描画 (ルートモジュール)
- `servo_sim.py`: シミュレーション実行用エントリポイント。各モデルの初期化と実行を管理。
- `sample_sim_core.py`: **物理エンジン本体**。PID制御器、LPF、1慣性/2慣性物理モデルの実装。
- `analysis_drawing.py`: 高品質なグラフ描画（Dark Mode対応）および統計量（オーバーシュート、制定時間）の等出。

## 📈 主な機能

1.  **マルチページ構成**: 目的（デモ調整 vs 自動エージェント）に応じたインターフェースの切り替え。
2.  **高度な物理シミュレーション**: 軸剛性や粘性摩擦を考慮した2慣性系の挙動再現。
3.  **マルチモーダルAIチャット**: 制御波形画像をチャットに挿入し、視覚的な情報を含めた調整相談が可能。
4.  **ワンクリック最適化**: 目標値への追従性能が最大化されるよう、PIDゲインを自動探索・反映。
